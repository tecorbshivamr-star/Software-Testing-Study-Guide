# Sprint Testing Activities

To prevent last-minute stress, QA testing activities must be distributed evenly across the 2-week sprint window.

---

## The Sprint Testing Checklist

### 📅 Days 1–2: Requirements & Planning
- [ ] Attend Sprint Planning.
- [ ] Review all sprint user stories.
- [ ] Clarify acceptance criteria using BDD/Gherkin if possible.
- [ ] Create high-level Test Scenarios for each card.

### 📅 Days 3–5: Test Design & Prep
- [ ] Write detailed Test Cases.
- [ ] Review test cases with Developers to ensure correct scope.
- [ ] Seed required Test Data in the QA database.
- [ ] Set up the test environments.

### 📅 Days 6–8: Test Execution
- [ ] Execute test cases as soon as developers deploy individual features.
- [ ] Run automated smoke tests on every deployment.
- [ ] Log defects in Jira with complete step-by-step details.
- [ ] Perform exploratory testing on finished features.

### 📅 Days 9–10: Regression & Closure
- [ ] Retest and verify resolved bugs.
- [ ] Run full automated Regression suites to ensure no new defects broke old features.
- [ ] Give QA Sign-off on the sprint stories.
- [ ] Attend the Sprint Review and Retrospective.

---

## 📌 Key Takeaways
- **Shift Left:** Start writing test cases on Day 1, not Day 8.
- **Continuous Execution:** Don't wait for all stories to be done before you start testing. Test features incrementally as they are deployed.
- **Defect Retesting:** Prioritize retesting resolved bugs early in the second week.
