# 🤝 Contributing to Software Testing Study Guide

First of all, thank you for considering contributing to this project!

The goal of this repository is to build the most comprehensive open-source Software Testing knowledge base for QA professionals, testers, developers, and students worldwide.

Every contribution matters, whether it's fixing a typo, improving documentation, adding new testing concepts, or sharing real-world testing examples.

---

# 📌 Ways to Contribute

You can contribute by:

- Adding new Software Testing topics
- Improving existing documentation
- Fixing spelling or grammar mistakes
- Sharing real-world testing scenarios
- Adding interview questions and answers
- Creating templates and artifacts
- Adding automation examples
- Improving project structure
- Updating outdated information

---

# 🚀 Getting Started

## 1. Fork the Repository

Click the **Fork** button at the top right of this repository.

---

## 2. Clone Your Fork

```bash
git clone https://github.com/your-username/software-testing-study-guide.git
```

---

## 3. Create a Branch

```bash
git checkout -b feature/topic-name
```

Examples:

```bash
git checkout -b feature/api-testing-guide
```

```bash
git checkout -b feature/interview-questions
```

---

## 4. Make Your Changes

Add your content following the existing folder structure.

Example:

```text
08_API_Testing/
└── 10_graphql_testing.md
```

---

## 5. Commit Changes

```bash
git add .
git commit -m "Add GraphQL testing guide"
```

---

## 6. Push Changes

```bash
git push origin feature/topic-name
```

---

## 7. Open a Pull Request

Create a Pull Request describing:

- What was added
- Why it was added
- Any references used

---

# 📝 Content Guidelines

Please follow these guidelines:

## Documentation

- Use Markdown (.md)
- Use clear headings
- Include practical examples
- Keep content beginner-friendly
- Use proper formatting

## Naming Convention

Use lowercase file names with underscores.

Example:

```text
01_what_is_software_testing.md
02_sdlc_vs_stlc.md
03_test_plan_guide.md
```

---

# 📂 Folder Structure

Place content in the appropriate section.

Examples:

| Topic | Folder |
|---------|---------|
| Selenium | 09_Automation_Testing |
| API Testing | 08_API_Testing |
| Agile | 07_Agile_Testing |
| Security | 11_Security_Testing |
| Interview Questions | 18_Interview_Preparation |

# 🔍 Quality Standards

All contributions should:

- Be easy to understand
- Include practical examples when possible
