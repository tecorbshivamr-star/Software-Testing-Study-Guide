# API Authentication Testing

Securing data is paramount. QA must know how to test and validate different API security and authentication mechanisms.

---

## 4 Common API Authentication Types

### 1. Basic Authentication
Uses a simple, base64-encoded username and password combination sent directly in the headers.
- **Header Structure:**
  `Authorization: Basic dXNlcm5hbWU6cGFzc3dvcmQ=`
- **Security Alert:** Extremely weak unless wrapped inside HTTPS (SSL/TLS). Base64 can be decoded in seconds.

---

### 2. Bearer Tokens & JWT (JSON Web Tokens)
The client sends credentials to a login endpoint. The server returns a JWT token. The client includes this token in the header of all subsequent requests.
- **Header Structure:**
  `Authorization: Bearer <JWT_Token>`
- **Understanding JWT Structure:**
  A JWT contains three parts separated by dots: `Header.Payload.Signature`
  - *Header:* Alg and type (e.g., HS256).
  - *Payload:* Claims (e.g., user ID, permissions, expiry date).
  - *Signature:* Verifies token integrity.

---

### 3. API Keys
A simple static string code generated by the server that the developer passes in headers or query parameters (e.g., `?api_key=xyz`).
- **Use Case:** Public services like weather APIs or maps.
- **Security Alert:** Easy to leak in URLs, source code, or browser logs.

---

### 4. OAuth 2.0
An industry-standard delegation framework that allows applications to access user data securely without exposing passwords (e.g., "Sign in with Google").
- **Workflow:**
  1. App requests authorization.
  2. User approves.
  3. App receives authorization code.
  4. App exchanges code for an **Access Token**.
  5. App queries API using the access token.

---

## QA Authentication Verification Checklist
- [ ] Verify that requesting endpoints without credentials returns `401 Unauthorized`.
- [ ] Verify that using an **expired** token returns `401 Unauthorized`.
- [ ] Verify that modifying a token's payload (altering the signature) returns `401 Unauthorized`.
- [ ] Verify **Role-Based Access (RBAC)**: An authenticated regular user trying to access admin endpoints gets `403 Forbidden`.

---

## 📌 Key Takeaways
- **401 Unauthorized** = You are not identified (invalid/missing token).
- **403 Forbidden** = You are identified, but you do not have permission.
- Always use HTTPS so credentials are not sent in plain text.
