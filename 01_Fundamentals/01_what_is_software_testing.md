# What Is Software Testing?

## Definition

**Software Testing** is the process of evaluating and verifying that a software product or application does what it is supposed to do. It involves executing a system in order to find defects, ensure it meets requirements, and deliver a quality product to users.

> *"Software testing is not just about finding bugs — it's about building confidence in the product."*

---

## Why Is Testing Necessary?

| Reason | Explanation |
|--------|-------------|
| **Find Defects Early** | Bugs found early cost 10–100x less to fix than in production |
| **Business Risk** | Faulty software can cause financial loss, legal issues, reputation damage |
| **User Satisfaction** | Quality software leads to better user experience and retention |
| **Regulatory Compliance** | Industries like healthcare and finance require certified quality |
| **Security** | Untested software is vulnerable to cyberattacks |

### Real-World Impact of Poor Testing
- 🚀 **NASA Mars Orbiter (1999)** — Crashed due to unit mismatch (metric vs imperial). Cost: $327 million
- 💸 **Knight Capital (2012)** — Faulty trading software lost $440 million in 45 minutes
- 🏥 **Therac-25 (1985–87)** — Radiation therapy machine software bug caused patient deaths

---

## Goals of Software Testing

1. **Defect Detection** — Find bugs before users do
2. **Quality Assessment** — Measure software quality against requirements
3. **Risk Mitigation** — Identify and reduce risks before release
4. **Confidence Building** — Give stakeholders confidence to release
5. **Compliance Verification** — Ensure regulatory standards are met

---

## What Does a Tester Do?

```
Understand Requirements
        ↓
Create Test Plans & Test Cases
        ↓
Set Up Test Environment
        ↓
Execute Tests
        ↓
Report Defects
        ↓
Verify Fixes (Retest)
        ↓
Sign Off on Release
```

---

## Testing vs Debugging

| Aspect | Testing | Debugging |
|--------|---------|-----------|
| **Done by** | QA Engineers | Developers |
| **Goal** | Find defects | Fix defects |
| **Process** | Planned, systematic | Investigative |
| **Output** | Defect reports | Fixed code |

---

## Key Terms

| Term | Meaning |
|------|---------|
| **Bug / Defect** | A flaw in the software |
| **Test Case** | A set of conditions to verify a feature |
| **Test Suite** | A collection of test cases |
| **Test Plan** | Document outlining the testing approach |
| **Test Environment** | Hardware/software setup for testing |
| **UAT** | User Acceptance Testing |

---

## 📌 Key Takeaways

- Testing ≠ Quality Assurance (QA is broader — process improvement)
- 100% testing is **impossible** — we test based on risk
- Testing can show **presence** of bugs but NOT their **absence**
- Testing is a skill that requires domain knowledge, creativity, and analytical thinking
