# QA Resume Building Tips

> A strong resume gets you past the ATS (Applicant Tracking System) and lands you the interview.

---

## 1. Structure and Format

- **Length:** Keep it to 1 page for junior/mid-level, maximum 2 pages for senior roles.
- **Format:** Always export as a standard PDF so formatting doesn't break.
- **Font:** Use clean, professional fonts (Arial, Calibri, Helvetica). No fancy graphics or progress bars (ATS systems can't read them).

**Standard Sections Order:**
1. Header (Name, Contact, LinkedIn, GitHub portfolio link)
2. Professional Summary (Optional, but good for context)
3. Technical Skills (Crucial for ATS keywords)
4. Work Experience (Most recent first)
5. Projects (Highly recommended for freshers/juniors)
6. Education & Certifications (e.g., ISTQB)

---

## 2. Technical Skills Section (The ATS Magnet)

Group your skills logically so recruiters can scan them quickly.

- **Testing Types:** Manual, Automation, API, Performance, Security, Agile/Scrum
- **Automation Tools:** Selenium WebDriver, Playwright, Cypress, Appium
- **Languages:** Python, Java, JavaScript, SQL
- **API Tools:** Postman, REST Assured, Swagger
- **Test Management/Defect:** Jira, TestRail, Zephyr
- **CI/CD & Version Control:** Git, GitHub Actions, Jenkins, Docker

---

## 3. Writing Impactful Experience Bullets (The XYZ Formula)

Avoid writing your resume like a job description ("Responsible for writing test cases"). Instead, write about your **Impact and Achievements**.

Use Google's XYZ formula:
> *"Accomplished [X] as measured by [Y], by doing [Z]."*

**Weak:**
- "Wrote automated tests using Selenium."
- "Found bugs in the application."
- "Tested APIs using Postman."

**Strong (XYZ Formula):**
- "Reduced regression testing time by **40%** (X) by developing a data-driven UI automation framework (Z) using **Selenium WebDriver and Pytest**."
- "Identified and logged over **150+ critical defects** (Y) across 4 major release cycles, improving production stability by **25%** (X)."
- "Automated **50+ REST API endpoints** (Z) using **Postman and Newman**, integrating them into the **Jenkins CI/CD pipeline** (Y) for nightly execution."

---

## 4. Key Action Verbs to Start Bullets

Never start a bullet with "Helped" or "Responsible for". Use strong action verbs:
- **Spearheaded, Architected, Developed** (For frameworks/plans)
- **Automated, Integrated, Optimized** (For efficiency improvements)
- **Identified, Resolved, Validated** (For testing activities)
- **Mentored, Led, Collaborated** (For leadership/teamwork)

---

## 5. Tips for Freshers / Career Changers

If you don't have professional QA experience yet, you *must* have a Projects section.

**Project Example:**
> **E-Commerce Automation Framework | Personal Project**
> - Developed a Hybrid Page Object Model (POM) automation framework using **Selenium and Python** to test an open-source e-commerce site.
> - Automated end-to-end flows including User Registration, Cart Management, and Checkout.
> - Integrated **Allure Reports** to generate visual test execution summaries.
> - *GitHub Link: [Link to your repo]*

*(If you don't have a project to link to, use the concepts in the [Real World Projects](../16_Real_World_Projects/) folder of this guide to build one!)*

---

## 📌 Final Checklist Before Applying
- [ ] Are there typos or grammar mistakes? (Use Grammarly).
- [ ] Are all my links (LinkedIn, GitHub) clickable?
- [ ] Did I tailor my skills section to match the job description keywords?
- [ ] Are my achievements quantifiable (do they include numbers/percentages)?
