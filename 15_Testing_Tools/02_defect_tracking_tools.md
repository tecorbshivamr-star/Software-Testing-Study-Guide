# Defect Tracking Tools

Defect tracking tools are essential for logging, prioritizing, and managing the lifecycle of software bugs from discovery to resolution.

---

## Top Defect Tracking Tools

### 1. Jira Software
- **Overview:** The industry standard for Agile project management and bug tracking.
- **Key Features:** Highly customizable workflows, Scrum/Kanban boards, extensive marketplace integrations.
- **Best For:** Almost any modern software development team.

### 2. Bugzilla
- **Overview:** A classic, open-source bug tracking system developed by Mozilla.
- **Key Features:** Extremely stable, lightweight, email notifications, and completely free.
- **Best For:** Open-source projects or teams needing a free, robust, no-frills tracker.

### 3. Azure Boards (Azure DevOps)
- **Overview:** Microsoft's Agile planning and defect tracking tool.
- **Key Features:** Kanban boards, backlogs, team dashboards, and seamless integration with Git and Azure Pipelines.
- **Best For:** Microsoft-centric development shops.

### 4. GitHub Issues / GitLab Issues
- **Overview:** Built-in issue trackers living alongside the source code repository.
- **Key Features:** Lightweight, developers can close issues directly via commit messages (e.g., `Fixes #102`), label management.
- **Best For:** Open-source projects, smaller teams, or developer-heavy workflows where simplicity is preferred over complex workflows.

---

## 📌 Key Takeaways
- Regardless of the tool, a defect must move through a structured lifecycle (New $\to$ In Progress $\to$ In QA $\to$ Closed).
- Jira is the undisputed king of enterprise defect tracking; mastering its JQL (Jira Query Language) is a highly valued QA skill.
